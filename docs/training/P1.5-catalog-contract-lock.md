### Listing Card Field Set (P2A canonical)

Every catalog card in P2A **must** render from exactly this field set:

```
id, slug, title, short_description, type, duration,
training_level_term_id, learning_format_term_id,
thumbnail_url, credential_eligible
```

The query **must also select** `duration_hours` and `created_at` for sorting, but these are **never rendered** on cards.

Taxonomy term IDs must be resolved to display names via a join to `taxonomy_terms.name`.

## 2. Semantic Field Contract

| Field | Semantic Role | Used In |
|---|---|---|
| `short_description` | One-line card summary (≤160 chars recommended) | Listing cards |
| `overview` | Multi-paragraph introduction to the item | Detail page hero |
| `description` | Full long-form content / syllabus prose | Detail page body |
| `duration` | Human-readable time string (e.g. "6 hours", "4 weeks") | Display everywhere |
| `duration_hours` | Numeric sortable value | `ORDER BY` clauses only |

### Delivery Mode vs Learning Format

- **`learning_format_term_id`** (taxonomy domain: `learning_formats`) is the **only** delivery descriptor used in P2A.
- The legacy `delivery_modes` domain is **deprecated and excluded** from P2A.
- No separate "delivery mode" field is required. `learning_format_term_id` covers: Self-Paced, Cohort-Based, Blended, Live Online, Mentor-Led.

## 3. Listing Eligibility Contract

A `training_items` row appears in catalog listings **if and only if** both conditions are met:

```sql
WHERE status = 'published'
  AND is_active = true
```

No other visibility rules apply in P2A. There is no soft-delete, no scheduling, no audience gating.

## 5. Sorting Contract for P2A

| Sort Option | P2A Status | Column | Direction | Sort Key |
|---|---|---|---|---|
| Recommended | ✅ Default | `created_at` | `DESC` | `recommended` |
| Newest | ✅ Included | `created_at` | `DESC` | `newest` |
| A → Z | ✅ Included | `title` | `ASC` | `title-asc` |
| Duration | ✅ Included | `duration_hours` | `ASC NULLS LAST` | `duration` |
| Oldest | ❌ Excluded | — | — | — |
| Duration descending | ❌ Excluded | — | — | — |
| Featured | ❌ Excluded | — | — | — |
| Relevance | ❌ Excluded | — | — | — |

### Default Sort Order

- **Public catalog pages** (`/training/*`): `recommended` (= `created_at DESC`)
- **School catalog page** (`/app/school/training/catalog`): `recommended` (= `created_at DESC`)

> "Recommended" currently aliases to newest. A scoring algorithm may replace it in a later phase without changing the sort key name.

## 6. Route-to-Type Contract

| Route | Type Filter | Behavior |
|---|---|---|
| `/training/courses` | `type = 'course'` | Shows courses only |
| `/training/packages` | `type = 'package'` | Shows packages only |
| `/training/pathways` | `type = 'pathway'` | Shows pathways only |
| `/training` | Mixed | Aggregates all types + static hero sections |
| `/app/school/training/catalog` | Mixed (default) | Shows all types; `type` filter is available |

### School Catalog Decisions

- The school catalog shows **all types by default** (no pre-selected type).
- The `type` filter **is available** as a selectable filter chip.
- No type is default-selected; the user may narrow by type optionally.

## 7. Final Locked Decisions

1. **Listing Card Field Set** is locked: `id, slug, title, short_description, type, duration, training_level_term_id, learning_format_term_id, thumbnail_url, credential_eligible` + sort-only fields `duration_hours, created_at`.
2. **Listing Eligibility** is locked: `status = 'published' AND is_active = true`.
3. **P2A Filters** are locked: `type`, `skills`, `competency_domains`, `curricula`, `grade_bands`, `subjects`, `learning_format`, `training_level`. No others in P2A.
4. **P2A Sort Options** are locked: `recommended`, `newest`, `title-asc`, `duration`. Default = `recommended`.
5. **Route-to-Type** is locked: typed routes use exact `type =` filter; `/training` and school catalog are mixed.
6. **Delivery mode is excluded** from P2A; `learning_format_term_id` is the sole delivery descriptor.
7. **Text search** uses `ILIKE` on `title` + `short_description`. No tsvector in P2A.
8. **Taxonomy resolution**: all `*_term_id` fields must be joined to `taxonomy_terms.name` for display.
