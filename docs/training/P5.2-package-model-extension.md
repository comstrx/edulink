# Phase 5.2 — Package Model Extension Decision Note

> **Status**: FROZEN | **Date**: 2026-03-13 | **Sprint**: P5.2

---

## Purpose

Document how Package fields map to the existing `training_items` schema, resolve modeling gaps, and confirm what was intentionally not implemented.

---

## A. Schema Changes Made

Four new columns added to `training_items`:

| Field | Type | Nullable | Default | Purpose |
|---|---|---|---|---|
| `pricing_type` | text | yes | NULL | Commercial pricing mode (fixed, custom, contact_sales, included_in_plan) |
| `price_amount` | numeric(12,2) | yes | NULL | Monetary price, meaningful only when pricing_type = 'fixed' |
| `price_currency` | text | yes | 'USD' | ISO currency code |
| `target_segment_term_ids` | uuid[] | yes | '{}' | Taxonomy IDs for target audience segments |

**Bundled courses** use the pre-existing `training_package_items` junction table (relational integrity) rather than a uuid[] array column.

**DB-level validation trigger** `validate_training_item_pricing` enforces:
- `pricing_type` must be one of: fixed, custom, contact_sales, included_in_plan
- `price_amount` must be >= 0 if set

**GIN index** added on `target_segment_term_ids` for efficient filtering.

---

## B. Validation Rules Added

Package-specific validation in `src/lib/training/package-form-schema.ts`:

- `bundled_course_ids`: array of valid UUIDs, managed via junction table
- `pricing_type`: enum (fixed | custom | contact_sales | included_in_plan), nullable
- `price_amount`: numeric >= 0, **required** when pricing_type = 'fixed'
- `price_currency`: max 3 chars (ISO code), nullable
- `target_segment_term_ids`: array of valid UUIDs, default empty
- No free-text bundled items
- No free-text pricing modes
- No self-referencing packages

Application-layer enforcement (via `usePackageCourses.ts`):
- `syncPackageCourses()` manages junction table — delete-then-insert with sort order
- `useAvailableCourses()` only returns `type = 'course'` items for selection

## D. Semantic Conflicts & Modeling Gaps

### 1. `bundled_course_ids` array vs `training_package_items` junction table

**Decision**: Use the existing `training_package_items` junction table.

The junction table provides:
- Referential integrity via foreign keys
- Sort order preservation
- No array-based FK violations
- Clean relational querying

The form uses a `bundled_course_ids` field internally, synced to the junction table on save.

### 2. Currency: taxonomy vs controlled text

**Decision**: Use controlled text field (`price_currency`), NOT a taxonomy UUID.

No currency taxonomy domain exists in the platform. Creating one for 3-letter ISO codes would be over-engineering. A controlled select with common currencies (USD, EUR, GBP, EGP, AED, SAR) is used. This is explicitly documented as a temporary approach.

**Gap**: If a canonical `currencies` taxonomy domain is created later, `price_currency` should be migrated to `price_currency_term_id`.

### 3. Target segment taxonomy

**Decision**: `target_segment_term_ids` stores taxonomy UUIDs.

No dedicated `target_segments` taxonomy domain exists yet. The field is ready for any approved taxonomy domain (school types, seniority levels, role categories). The admin form does not yet expose a selector — this requires a canonical targeting taxonomy to be established first.

**Gap documented**: Target segment selector is deferred until a canonical targeting taxonomy is approved.

### 4. `price_amount` type: `numeric(12,2)` vs request's `numeric(12,2)`

**Decision**: Matches exactly. No conflict.

## F. Architectural Decisions

### Package composition rules
- A package may only bundle items where `training_items.type = 'course'`
- No package-inside-package
- No pathway-inside-package
- Enforced at application layer via `useAvailableCourses()` filter

### Pricing is metadata, not billing
- `pricing_type`, `price_amount`, `price_currency` are product representation only
- No checkout, invoicing, or payment integration
- No discount engine

### Canonical data flow
```
Package Metadata (training_items where type = 'package')
        ↓
Bundled Courses (training_package_items junction)
        ↓
[Future] Package Enrollment
        ↓
[Future] Course-level Progress
        ↓
[Future] Package Completion
```
